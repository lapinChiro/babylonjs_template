# ルール新設時の遡及適用

## 適用条件

`.claude/rules/*.md` または `.claude/skills/*/SKILL.md` を**新規作成**するとき。既存ルールの内容追記・軽微修正は対象外。

## 背景

ルールを新設する行為自体に満足して既存コードへの遡及適用を怠ると、そのルールが最初に防ぐべきだった既存の同型違反を見逃す。ルール制定と施行の乖離は組織学習の典型的失敗パターンであり、制定と同時に既存違反を掃討して初めてルールが機能する。

## 制約

ルールを新規作成する PR（または単一作業単位）には、**必ず以下の 2 点を同時に含める**こと:

1. **ルール本体** — 適用条件・制約・禁止事項・検証を記述した `.md` ファイル
2. **既存コードへの遡及適用の実行** — 本ルールの適用条件に該当する既存コードを grep/監査で網羅的に特定し、違反がある箇所を修正する。違反を発見したら黙って直さず、修正コミットで対応する（git history が遡及適用の透明な記録になる）

### 遡及適用を分割する場合

対象が多く同一 PR での修正が非現実的な場合のみ、以下を満たせば分割してよい:

- `TODO` ファイル（リポジトリルート）に未対応の対象を個別タスクとして起票する
- 分割理由を明示する（例: 「対象 30 件、単一 PR では review 困難のため 5 件ずつ 6 PR に分割」）

「後でやる」という曖昧な記述のみでは要件を満たさない。必ず追跡可能な形にする。

## 禁止事項

- ルールを新規作成した PR で、既存コードへの遡及適用を**一切行わない**こと
- 遡及適用を「後続 PR で扱う」と曖昧に記述するだけで TODO に起票しないこと

## 検証

- 新設された `.claude/rules/*.md` / `.claude/skills/*/SKILL.md` の適用条件に該当する既存コードが grep/監査で特定され、違反があれば同一 PR で修正されている（git log で追跡可能）
- 分割適用の場合は `TODO` に対応する起票が存在する
