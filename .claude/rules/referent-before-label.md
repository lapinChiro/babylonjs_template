# 語より先に指示対象を固定する（referent before label）

## 適用条件

次のいずれかに該当するとき。該当するか判断できない場合は該当扱いにする。

1. **文書級**: 設計資料・PRD・調査報告・原因切り分け案・対策案など、見出し構造を持つ文書を新規作成する、または章単位で書き換えるとき
2. **記録級**: TODO 起票・plan.md 追記など短い記録を書くとき、命名する（状態名・条件名・事象名・値や記録の型名・メソッド名・boolean 名）とき、ユーザーが与えた推論順序を短い作業ラベルへ要約しようとしているとき

対象外: ユーザー原文の引用、単純な機械編集（項目移動・削除・書式修正）、既存名の再利用、確立した用語だけで書ける短文。

ロード経路: path-scope（上記 glob の Read / Edit 時に自動ロード）。**新規ファイルの Write では発火しない**ため、文書級の入口 skill（`prd-template` / `investigation`）が起草前に本 rule の能動 Read を指示する。

## 背景

生成 AI は、対象や目的が定まっていなくても、もっともらしい語を置けば流暢に文章を続けられる。対象が曖昧なまま語（しばしばその場の造語）を先に置き、その語を土台に思考を進めると、指示対象とのずれが訂正されないまま設計文・コード識別子へ波及する。実例: 調査レポートの見出し語「圧縮時点」が開始条件・値・事象の三役を行き来し、調査結果が誤った方向へ誘導された。「観測を一つに集める」という要約で「試験 → 結果から切り分け → 対策選定」という推論順序が消えた。

誤用語の訂正リストとの照合（言葉狩り）は対策にならない。造語は列挙できず、次は別の造語が生まれるためである。止めるべきは「語を先に置き、対象を後付けする」生成順そのものであり、本 rule はその順序を制約する。

## 制約

### 1. 文書級では対応表（referent table）を本文より先に独立保存する

本文を書く前に、対応表を scratchpad（セッションの一時領域）へ `referent-table-<slug>.md` として保存する。**先に独立ファイルとして Write した事実**が「語より先に対象を固定した」記録になる（完成文書の先頭に表を置くだけでは生成順の証明にならない）。

書式（列順の変更禁止）:

| 出典 | 目的 | 具体対象 | 役割 | 前後関係 | 候補語 | 初出定義 |
|---|---|---|---|---|---|---|

- **候補語は右端に固定し、具体対象と役割が埋まるまで空欄のまま**にする（語を先に決めて対象を後付けする順序を、書式で不可能にする）
- 役割は閉じた選択肢から選ぶ: `開始条件 / 状態 / 事象 / 値 / 記録 / 目的 / 手段`。同一語が複数役割を指す場合は行を分ける
- 「前後関係」にはユーザーが与えた推論順序を**原文の語**で書く（役割列と混ぜない）
- 表は通常 1〜6 行。超える場合は意味が変わる境界で表を分割する
- 完成時に、候補語列を隠しても「具体対象」列だけで意味が通ることを確認する

### 2. 候補語の埋め方（文書級・記録級とも）

- ユーザーの用語・確立した用語を最優先で使う
- それ以外の新語は、初出に「X とは〜を指す」の定義文を書けるときだけ導入する。定義が書けない語は導入せず、具体対象の記述をそのまま本文で使う

### 3. 本文は対応表の写しとして書く

- 文書級の本文の中心語彙は、対応表に載った語だけを使う
- 日本語文・設計要素・コード識別子の三層で同じ対応を保つ（例: 開始条件 → 条件名、事象 → 事象名・メソッド名、状態 → 状態名。**別役割は別名**にする）
- 骨組み・見出し・結論に**作業ラベル**（目的・対象・判断を省いたまま未整理の作業を抽象名詞で包んだ句。例: 「観測をまとめる」）を使わない。使いたくなったら、その句の指示対象を対応表に書けるか試し、書けなければ具体文で書く

### 4. 違反に気づいたときの復旧

- 対応表を保存せず文書級の本文を書き始めたことに気づいたら、後付けで表を追記して継続しない。**本文を破棄し**、対応表を独立保存し直してから本文を再生成する
- 表の行の誤り（指示対象・役割の取り違え）を指摘されたら、説明を追加するのではなく、該当行を書き直してから本文の該当箇所を再生成する

## 禁止事項

- 文書級の対象文書で、対応表を独立保存せずに本文を書き始めること
- 具体対象・役割が空欄のまま候補語列を先に埋めること
- 一語に複数役割（開始条件と事象など）を負わせたまま書き進めること（行を分けて別名にする）
- 骨組み・見出し・結論に作業ラベルを使うこと
- 初出定義を書けない新語を導入すること
- 表なしで書き始めた本文に、後から表を追記して「先に作った」ことにすること
- 誤用語リストとの照合を対策にすること（照合先はその文書自身が宣言した対応表）

## 検証

- 文書級の成果物について、scratchpad に対応する `referent-table-<slug>.md` が本文より先に保存されている
- 本文の見出し・中心語彙が対応表の候補語と一致し、初出定義のない新語がない
- 命名結果が役割と対応している（条件名・事象名・状態名が別名になっている）
