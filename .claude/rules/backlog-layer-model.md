# backlog 三層構造モデル

## 適用条件

`plan.md` / `backlog/` / `TODO` / `TODO.archive` のいずれかを変更するとき、これらの間の**構造的整合**を保つ。

本 rule は backlog 体系の**構造モデルと不変条件**（何が常に真でなければならないか）を定める。各層を操作する**具体手続き**（追加・移送・完了・退避の手順、`plan.md` 更新手順、命名規約）は `backlog-management` skill（`.claude/skills/backlog-management/SKILL.md`）を参照する。

## 制約

### 三層構造 + 退避層

| 場所 | 役割 | 扱う対象 |
|------|------|----------|
| `plan.md` | PRD 化済みタスクの消化順序 | `backlog/` 内の PRD の実行順序を管理。何を次にやるかを示す |
| `backlog/` | 設計済み・着手可能な PRD | 1 機能 = 1 ファイル。PRD テンプレートに従う |
| `TODO` | PRD 化前の課題・アイデア | 優先順位付けと PRD 化判定を行う inbox。何から PRD 化するかを示す |
| `TODO.archive` | **着手・精査したが、ユーザーが「対応見送り」と判断した課題の退避層** | 「現状維持で決着」「実施しない」「解決/クローズ済み」と**ユーザーが判断**したものを、削除せず記録として保持する。各項目は「再開条件 / 再発トリガー」を持ち、満たされたら `TODO` へ戻す |

### TODO 項目の出口は 2 つだけ（不変条件）

`TODO` に記載された各項目は、最終的に以下のいずれかで `TODO` を去る。第 3 の出口（黙って消す等）は禁止:

1. **PRD 化で解消** — `backlog/` に PRD として起票 → `plan.md` で消化順序決定 → 着手・完了 → 削除（解決）
2. **`TODO.archive` へ退避** — ユーザーが「対応見送り（現状維持 / 実施しない / 解決済み）」と判断 → `TODO.archive` へ転記（記録保持・再開条件付き）

「精査の上で現状維持と決着したか」は**ユーザーの判断**であり、エージェントが独断で archive しない（archive 範囲はユーザーに確認する）。

### パイプライン

```
TODO（課題発見）─┬─ PRD 化 → backlog/ → plan.md → 着手 → 完了 → 削除   （解消）
                 └─ 対応見送り（ユーザー判断） → TODO.archive            （退避・再開条件付き）
```

### `plan.md` の前向き不変条件

`plan.md` の本文は **「今後の開発計画」と「現在進行中の取り組み」だけ**を持つ（前向き）。後ろ向きな完了記録は末尾「完了 PRD（番号 + 短タグ・最新 10 件）」index に限定する（冒頭の自己宣言「本ファイルは今後の開発計画と現在進行中の取り組みを記載する」が正）。

- 完了痕跡は末尾 index の **番号 + 短タグ**のみ（消化順序・進行中セクションには残さない）
- index は **2 列テーブル（`| PRD | 短タグ |`）で最新 10 件のみを保持し、新しい順（最上段が最新）**に並べる。PRD 完了時に**最上段へ 1 行追記し、11 件目＝最古行を削除する**。10 件を超えて過去ログを溜めない（それ以前は plan.md の git 履歴を正とする）
- 短タグは想起の手がかりに留め、**実装詳細（関数名・スキーマ・カラム・分岐等）を列挙しない**（肥大化と drift の原因。正は git log / 完了コミット / report。`documentation-drift-prevention` 準拠）
- 同じ PRD のステータス（実装済・未コミット等）を スナップショット / 進行中 の複数箇所で重複記述しない（1 箇所を詳細・他は 1 行ポインタ）

### 層間の整合不変条件

- `backlog/` 内の全 PRD が `plan.md` に記載されている
- `plan.md` の消化順序・進行中の全項目が `backlog/` 内の PRD に対応している（末尾 index と保留・制約セクションを除く）
- `backlog/` に移した項目は `TODO` に重複して存在しない
- `TODO` から削除された項目は「`backlog/` に PRD 化された」か「`TODO.archive` に転記された」かのいずれかで追跡できる（情報の消失がない）
- `TODO.archive` の各項目に「再開条件 / 再発トリガー」が記載され、`TODO` 本文に重複しない
- backlog ファイル名で使った PRD 番号は再利用しない（git history と一致させるため。完了済番号も再利用不可）

### トラック（`plan.md` 内の前向き計画単位・任意）

**トラック** = 共通ゴールへ向けて連続的に消化される、順序付き PRD 群の**前向き計画単位**。`plan.md` の前向きセクションに置く。作成・保守・中断・終了の**具体手続き**は `track-management` skill（`.claude/skills/track-management/SKILL.md`）を参照する。

**トラックは任意（開発の 3 モード）**: 開発は以下いずれの形でもよく、トラックは必須でない —

1. **トラック駆動**: トラックを作り、その消化順序に沿って PRD を連続消化する
2. **単発 PRD**: `TODO` から 1 項目を取り出して PRD 化する（どのトラックにも属さない）
3. **その場 PRD**: ユーザー指示でその場で PRD 化する

単発・その場 PRD がどのトラックにも属さないことは正常（トラック所属を全 PRD に強制しない）。

**JIT PRD 不変条件**: トラックは単位分解と消化順序を `plan.md` に持つが、**PRD ファイルは「次に着手する 1 本（＋進行中）」だけ `backlog/` に材料化**する。未着手の後続単位は `plan.md` に**単位名 + 一言**で置き、着手直前に `prd-template` で PRD 化する。理由: 後続 PRD を前倒しで作り置きすると、先行 PRD の結果・学びが後続設計に反映されず陳腐化する。

**layer-model との関係**: トラックの前向きセクション（未材料化の後続単位）は**消化順序・層間整合の不変条件の対象外**（既存の前向き計画の扱いと同じ）。`backlog/` に材料化された PRD のみが厳密な消化順序・層間整合の対象になる。

## 禁止事項

- `TODO` 項目を「PRD 化」「`TODO.archive` 退避」のどちらも経ずに黙って削除すること（第 3 の出口は禁止）
- `plan.md` の**消化順序・進行中セクション**に完了済み項目を残すこと（打ち消し線・完了マーク・「完了」ラベル等。完了は末尾 index への 番号 + 短タグ 追記で表す）
- `plan.md` 末尾 index の短タグに**実装詳細**（関数名・スキーマ・カラム・分岐等）を列挙すること
- `plan.md` 本文に**完了イベントの履歴 prose**（完了報告・トラック消化完了等の後ろ向き記述）を残すこと
- `plan.md` に PRD 化前の情報（`TODO` の内容）を記載すること
- 完了した PRD ファイルを `backlog/` に残すこと / `backlog/` に移した項目を `TODO` に残すこと
- エージェントが**ユーザー判断を経ずに独断で** `TODO.archive` へ移すこと（候補の分類・提示までは可、移送はユーザー確認後）
- `TODO.archive` へ転記する際に元の記述内容・再開条件を欠落させること / 転記済み項目を `TODO` 本文に残すこと
- トラックの**未着手の後続単位を前倒しで `backlog/` に PRD ファイル化して作り置き**すること（JIT PRD 不変条件違反。後続単位は `plan.md` に単位名 + 一言で置き、着手直前に材料化する）
- 孤立した単発課題をトラック所属に**強制**すること（トラックは任意。単発・その場 PRD はトラックに属さなくてよい）

## 検証

- `plan.md` の消化順序・進行中セクションに完了済みの項目が存在しない（完了痕跡は末尾 index の 番号 + 短タグ のみ）
- `plan.md` 本文に完了イベントの履歴 prose が無い（前向き＝今後の計画 + 現在進行中のみ）
- 「完了 PRD」index が **2 列テーブル（`| PRD | 短タグ |`）・最新 10 件・新しい順**であり、各エントリが **番号 + 短タグ（実装詳細なし）**である（11 件以上溜まっていない）
- `plan.md` の消化順序・進行中の全項目が `backlog/` 内の PRD に対応している（index と保留・制約セクションを除く）
- `backlog/` 内の全 PRD が `plan.md` に記載されている
- `backlog/` に記載されている項目が `TODO` に重複して存在しない
- `TODO` から削除された項目が 2 つの出口のいずれかで追跡できる（情報の消失がない）
- `TODO.archive` の各項目に「再開条件 / 再発トリガー」が記載され、`TODO` 本文に重複しない
- `backlog/` に「次に着手する 1 本（＋進行中）」を超える**未着手 PRD の作り置きが無い**（JIT PRD 不変条件）
- トラックの未着手の後続単位が `plan.md` に**単位名 + 一言**で置かれ、PRD ファイル化されていない
