# 検証の原則

## 適用条件

手動確認・自動テスト設計・品質チェックなど、何かを検証する場面すべて。

## 制約

検証は「実行して結果を見る」行為ではなく、「計画 → 実行 → 判定」のプロセスである。

1. **検証項目の列挙**: 何を検証するかを事前に MECE に洗い出す。定型のチェックリストは最低限の floor とし、対象固有のリスクから検証項目を動的に追加する（[`coverage-floor-not-ceiling.md`](coverage-floor-not-ceiling.md)）
2. **期待結果の定義**: 各項目について、どのような結果なら合格/不合格かを実行前に決める
3. **実行**: 検証を実行する
4. **判定**: 事前に決めた基準に照らして結果を評価する

## 禁止事項

- 実行して出力を「見る」だけで確認完了とすること
- 正常系だけ確認して異常系を確認しないこと
- 結果が出てから「これは期待通りだ」と後付けで判断すること
- 検証項目を洗い出さずに「とりあえず動かしてみる」こと
- **検証コマンドの "0 errors" 出力だけを信頼すること**: 出力の数値 (test count 等) も検算対象として「想定通り」を確認する（local check を CI と同条件にする具体機構・不変条件は `.claude/rules/local-quality-mirrors-ci.md` を正とする）
- **local check が CI と同条件で動いていることを保証せずに完了報告すること**: 具体手順・不変条件は `.claude/rules/local-quality-mirrors-ci.md` を参照
