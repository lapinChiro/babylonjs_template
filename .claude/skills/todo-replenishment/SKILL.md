# TODO の補充

## トリガー

`TODO` が空（未整理のアイデアがない）の状態でユーザーから作業依頼を受けたとき。

## アクション

1. 現在の実装を分析する（機能一覧、テストカバレッジ、既知の制約、技術的負債）
2. プロジェクトの目的（Web 3D アプリケーションテンプレート: Vue 3 + Babylon.js + Hono + PostgreSQL + MinIO）を考慮し、次に価値のある機能・改善を提案する
   - 未実装の機能（ユーザー体験の向上）
   - 品質改善（テスト、エラーハンドリング、パフォーマンス等）
   - 開発基盤の整備（CI、DX、リファクタリング）
3. ユーザーにヒヤリングし、優先度と方向性を確認する
4. 合意した項目を `TODO` に書き込む

## 禁止事項

- ユーザーへのヒヤリングをせずに `TODO` に項目を書き込むこと
- 分析結果を提示せずに「何をやりましょうか？」とだけ聞くこと（具体的な提案を伴うこと）
