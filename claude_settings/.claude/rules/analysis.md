# 分析の原則

## 適用条件

対象を分析する（理解のために観点で評価し、観点そのものを獲得していく）とき。調査(`investigation`)・レビュー(`review-dev` / `review-prd`)・テスト設計(`test-design`)・設計評価など、対象を多角的に評価して理解・判断を得る場面すべて。

検証(`verification`: 既定基準への合否確認)とは別の行為 — 分析は観点を**生成**して対象を**理解**する探索的行為であり、検証はその後段で確定基準に照らす確認的行為。

## 制約

分析は「思いついた観点で評価する」一発勝負ではなく、**問いで有界化された螺旋**として行う:

0. **問いの固定**: 何のための分析かを先に固定する。観点空間は問いがないと無限で、MECE も停止も定義できない。問いが枠を与えて初めて観点が有界になる。
1. **評価**: ある観点で対象を評価する。
2. **観点の獲得（内生＋外生）**: 評価結果から次の観点を導く。内生導出だけだと既存観点の近傍に偏り経路依存の死角が残るため、**定期的に外生の観点を注入**する（確立した枠組み・第三者視点・「反証せよ」と指示した敵対的レンズ）。
3. **観点マップの収束**: 新観点が枯れるまで 1–2 を反復する。**MECE は宣言・証明できない**（unknown unknowns）。「何が漏れているか」を専任で問い（completeness critic）、K 回連続で新観点が出ないことをもって漸近的に収束とみなす。
4. **マップで再評価しループバックする**: 得た観点マップで対象を評価する。観点は**問いへの関連度・リスクで重み付け**する（全観点等価でない）。再評価で出た新観点は 2 へ戻す（4 は終点でなく螺旋の一周）。

観点の floor（固定列挙）からの動的拡張・対象固有リスクからの導出は [`coverage-floor-not-ceiling`](coverage-floor-not-ceiling.md) に従う。

## 禁止事項

- 問いを固定せずに観点を挙げ始めること（観点空間が無限になり収束しない）。
- 最初に思いついた観点セットだけで評価を終えること（螺旋を回さない一発分析）。
- 内生導出のみで観点を広げ、外生レンズ・反証視点を注入しないこと（経路依存の死角を放置）。
- 「MECE な観点で見た」と網羅を宣言すること（MECE は証明できない。completeness critic と枯れ確認で漸近させる）。
- 全観点を等価に扱い、問いへの関連度・リスクで重み付けしないこと。
- 再評価で出た新観点を捨て、螺旋をループバックさせずに打ち切ること。
